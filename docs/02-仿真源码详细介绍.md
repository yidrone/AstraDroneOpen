# 仿真源码详细介绍

本文档旨在介绍项目中与仿真有关的内容。

## 1  Simulation资源清单

**AstraDroneOpen/simulation/astra_gazebo_models:**其中包含AstraSim提供的诸多仿真模型，总计。每个模型已经链接到gazebo中，可以直接进行测试与实验。

**AstraDroneOpen/simulation/astra_gazebo_worlds：**提供了仓库、森林等数种世界文件以供测试与示例使用。

**AstraDroneOpen/simulation/px4_sim_files:**提供与px4紧密联系的机型相关文件，包括机架参数文件、机架sdf文件以及配套的仿真无人机gazebo启动文件。 

本目录提供去除GPS后的机型、携带mid360的机型以及携带深度相机的机型，以供仿真测试与使用。

### 2.2  程序包介绍

#### 2.2.1  sensors/

本目录主要提供了传感器的相关源码，主要包括gazebo插件与模型描述文件。

**lidar_simulator/lidar_discription**：提供雷达的基本仿真，提供雷达的描述文件等。如：`custommsg`定义、各种雷达的`urdf`文件、`meshes`描述文件以及相关的启动文件。为雷达在gazebo中模型和结构的显示与连接提供支持。

**lidar_simulator/lidar_gazebo_plugins**：提供雷达的gazebo插件，可以理解为传统机械雷达在仿真中的驱动，以便雷达可以在仿真中发出相应的点云等话题。在启用lidar时，需要在gazebo的插件路径找到描述文件中规定的插件，如`livox`雷达中就是`liblivox_laser_simulation.so`需要被读取。

**Mid360_simulation_plugin/livox_laser_simulation/**：主要提供对Mid360雷达的驱动。本部分源码是在gazebo仿真中雷达相关点云话题生成的关键源码部分。其中 **scan/** 包含了livox不同型号雷达的扫描配置，编译结果中有livox雷达的关键仿真插件。

#### 2.2.3  gazebo_sim/

本目录主要提供无人地面车辆(ugv)的相关仿真文件。

总共提供两种车型，一种为`hunter2`，一种为`hunter EC`。

**hunter/hunter2_base**：里面包括无人车的各个关节(`joint`)名字，以及提供无人车的外观模型以及碰撞(`collision`)模型，包括各个连杆结构等，还有无人车的urdf骨架文件

**hunter/hunter2_control**：包括基本的控制器，允许导航模块向无人车发布控制指令，并将ackermann控制指令传达到电机层次。

`keyboard_teleop.py`：建立一个基本的控制指令发布，可以通过键盘的wasd发布控制指令

`keyboard_teleop.py`中，通过ackermann控制指令的回调函数中：

```python
def set_throttle_steer(data):

    global flag_move

    pub_vel_left_rear_wheel = rospy.Publisher('/hunter2/wheel_joint1_velocity_controller/command', Float64, queue_size=1)
     ...
    pub_pos_right_steering_hinge = rospy.Publisher('/hunter2/wheel_joint4_steering_position_controller/command', Float64, queue_size=1)

    throttle = data.drive.speed/0.1
    steer = data.drive.steering_angle

    pub_vel_left_rear_wheel.publish(throttle)
	...
    pub_pos_right_steering_hinge.publish(steer)
```

将ackermann的控制指令转移到具体的电机与舵机信号，进而控制无人车。

**hunter/hunter2_gazebo**：提供了`ugv`在`gazebo`中的启动文件，包括相应的世界文件。通过直接获取`gazebo`的定位消息作为里程计消息的Python脚本，获取完美的地面真值。同时还包括了相应的世界文件和提供无人车描述的`xacro`文件。

**hunter/steer_bot_hardware_gazebo**：控制器的硬件抽象层，连接编写的指令发布器和gazebo内部模型的指令接受，以便ugv能够在仿真环境中完成相应的指令控制。

阿克曼转向几何计算，代码在这里手动实现了阿克曼转向原理，而不是依赖 Gazebo 的插件。

```c++
if(enable_ackermann_link_) {
  // 内轮转角 > 外轮转角
  // 计算左/右前轮所需的独立转向角
  pos_cmd = atan2(2*h*tan(front_steer_jnt_pos_cmd_), 2*h + w/2.0*tan(front_steer_jnt_pos_cmd_));
  // ... (另一侧同理，只是分母变减号)
}
```

通过readSim,读取 Gazebo 中左、右两个舵机的实际角度，取**平均值**，伪装成一个单一的“前轮转向角”反馈给 ROS。这保证了上层算法不会因为看到两个不同的轮子角度而困惑。完成了算法层面到物理层面的交接。

**hunter/steer_drive_controller**：主要的控制器部分，完成了速度限制、转向驱动以及里程计，完成了基本的控制流程，保证指令的物理可实现性。

```C++
update(){
    const double linear  = rear_wheel_est_vel;
	const double angular = tan(front_steer_pos) * linear / wheel_separation_h_;	
}
```

实现解码器计算里程计。

对于hunter EC部分，基本结构相同。

**hunter_ec/hunter_se_gazebo**：提供无人车与gazebo的连接以及指令的连接

**hunter_ec/hunter_se_control**：提供无人车的控制器以及pid等参数文件。

**hunter_ec/hunter_se_description**：提供无人车的描述文件，包括外观、碰撞以及无人车结构骨架。以及相应的参数、启动文件与世界文件。
