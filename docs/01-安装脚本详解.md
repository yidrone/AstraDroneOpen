# **安装脚本使用教程**

此安装脚本支持PX4安装与初始化、相关源码编译与第三方库安装。支持**pc**和**onboard**两种环境的搭建。将不同的安装过程模块化，用户可以依据不同的需要来运行不同的脚本。对于新系统或新设备环境的配置提供支持。



本教程旨在介绍安装脚本的使用方法、环境安装与配置过程，以及部分已知的问题。

```terminal
cd ~/AstraDroneOpen/ && sudo chmod 777 ~/AstraDroneOpen/scripts/*
./scripts/pc_installer_x86.bin
```

> [!IMPORTANT]
>
> 在运行本脚本时一定要**注意输入密码**以授予sudo权限。
>
> wsl运用该脚本时要事先apt install一下**gnome-terminal**，以便后续的安装。

### 2 run_sh 其他脚本：

#### 1. 脚本名称：`demonstration.sh`

- **功能概述**: 启动一套完整的演示系统，包含底层驱动、感知定位、控制算法以及可视化工具，用于全面的无人机系统展示。
- **启动的主要模块/指令**:
  - **基础环境**: `roscore`, `rqt` (图形化调试工具)。
  - **飞控通信**: `mavros px4.launch` (连接 PX4 飞控)。
  - **传感器驱动**: `livox_ros_driver2` (Mid360 激光雷达), `realsense2_camera` (深度相机), CSI 摄像头脚本 (`start_camera_with_retry.sh`)。
  - **定位与建图**: `fast_lio` (基于激光雷达的 SLAM)。
  - **控制逻辑**: `astra_control` (无人机控制节点)。
  - **可视化**: `rviz` (加载了 `astra_pc.rviz` 配置文件的 3D 可视化界面)。

#### 2. 脚本名称：`echo.sh`

- **功能概述**: 专门用于调试和监控，实时打印无人机的状态和位置信息，不启动控制或驱动节点。
- **启动的主要模块/指令**:
  - **位置/姿态**: `rostopic echo /mavros/local_position/pose`。
  - **连接状态**: `rostopic echo /mavros/state`。
  - **位置设定点**: `rostopic echo /mavros/setpoint_position/local`。
  - **原始设定点**: `rostopic echo /mavros/setpoint_raw/local`。

#### 3. 脚本名称：`onboard_example.sh`

- **功能概述**: 适用于机载计算机的启动脚本，主要运行传感器驱动和定位算法，不包含高层控制逻辑或图形化界面。
- **启动的主要模块/指令**:
  - **基础环境**: `roscore`。
  - **飞控通信**: `mavros px4.launch`。
  - **传感器驱动**: `livox_ros_driver2` (激光雷达), `csi_camera` (CSI 摄像头), `realsense2_camera` (RealSense 相机)。
  - **定位与建图**: `fast_lio` (SLAM 算法)。

#### 4. 脚本名称：`pc_example.sh`

- **功能概述**: 适用于地面站 PC 端或仿真的脚本，包含地面站软件、离板控制逻辑以及对应的 PX4 启动文件。
- **启动的主要模块/指令**:
  - **基础环境**: `roscore`。
  - **仿真/飞控**: `roslaunch px4 astra_example.launch`。
  - **定位与建图**: `fast_lio` (配置为 `rviz:=false`，即不启动可视化)。
  - **控制逻辑**: `offboard autoarming_control.launch` (自动解锁与离板控制)。
  - **地面站**: `qgc` (QGroundControl)。

#### 5. 脚本名称：`record.sh`

- **功能概述**: 数据采集脚本，用于录制飞行过程中的关键传感器数据和状态信息到 rosbag 文件中。
- **启动的主要模块/指令**:
  - **数据录制**: `rosbag record`。
  - **录制的话题**: 包含压缩后的图像 (`/camera/.../compressed`), 激光雷达点云 (`/livox/lidar`), SLAM 结果 (`/cloud_registered_body`, `/freedom/static_pointcloud`), 飞控状态 (`/mavros/state`, `/mavros/battery`), 以及位置/里程计/TF 等信息。

#### 6. 脚本名称：`test.sh`

- **功能概述**: 用于实机飞行测试的精简脚本。与演示脚本类似，但移除了可视化工具 (Rviz, Rqt) 和 RealSense 相机，增加了一个状态回显窗口。
- **启动的主要模块/指令**:
  - **基础环境**: `roscore` 以及 `rostopic echo /mavros/local_position/pose` (用于监控位置)。
  - **飞控通信**: `mavros px4.launch`。
  - **传感器驱动**: `livox_ros_driver2` (激光雷达), CSI 摄像头脚本。
  - **定位与建图**: `fast_lio`。
  - **控制逻辑**: `astra_control`。

### 3 env_sh

#### 1.脚本名称：`01_env_px4_init_arm.sh`

- **适用平台**: ARM 端 (机载计算机，如 Jetson Orin/Xavier 或 Raspberry Pi)
- **功能概述**: 针对嵌入式设备的 PX4 环境配置脚本。与 PC 版相比，通常会精简掉图形化仿真界面（Gazebo UI），专注于通信和控制接口。
- **主要操作/安装内容**:
  - **MAVLink 依赖**: 安装 MAVROS 及其所需的地理数据集。
  - **编译优化**: 针对 ARM 架构可能的特定依赖项安装。
  - **环境精简**: 可能跳过仅在桌面端需要的 SITL (软件在环仿真) 组件。

#### 2.脚本名称：`02_env_third_party_init_arm.sh`

- **适用平台**: ARM 端 (机载计算机)
- **功能概述**: 在机载电脑上部署第三方驱动和库，确保实机飞行时的传感器正常工作。
- **主要操作/安装内容**:
  - **硬件特定驱动**: 可能会从源码编译适合 ARM 架构的 `librealsense` 或 CSI 摄像头驱动。
  - **性能配置**: 可能包含针对 Jetson 平台的 CUDA 配置或 NVP Model 设置。
  - **算法库**: 安装与 PC 端相同的数学库，但针对 ARM 处理器进行编译。

#### 3. 脚本名称：`checkout_cmake_versions.sh`

- **功能概述**: 版本控制辅助脚本，用于检查或切换当前系统中的 CMake 版本，或检查项目依赖库的 CMake 配置。
- **主要操作/安装内容**:
  - **版本检查**: 验证 `cmake --version` 是否符合项目构建要求（某些 ROS 包或 Fast-LIO 可能需要较高版本的 CMake）。

#### 4. 脚本名称：`cmake-install.sh`

- **功能概述**: 自动化安装指定版本 CMake 的脚本。
- **主要操作/安装内容**:
  - **下载与解压**: 从 CMake 官网或 GitHub 下载特定版本的源码或二进制包。
  - **路径配置**: 将新版本的 `cmake` 二进制文件链接到 `/usr/bin` 或 `/usr/local/bin`，覆盖系统默认的旧版本，以解决编译兼容性问题。

#### 5. 脚本名称：`ignore_packages.sh`

- **功能概述**: 编译优化工具，用于在大规模工作空间中忽略不需要编译的 ROS 包。
- **主要操作/安装内容**:
  - **创建标记**: 在指定的功能包目录下创建 `CATKIN_IGNORE` 空文件。
  - **作用**: 告诉 catkin 编译系统跳过这些包，从而加快编译速度（例如在仅调试控制算法时，忽略视觉处理相关的包）。

### tools

#### 1. 脚本名称：`bag_to_video_and_images.py`

- **文件类型**: Python 脚本 (依赖 ROS, OpenCV)
- **功能概述**: 这是一个数据处理工具，用于从 ROS 录制的数据包 (`.bag` 文件) 中提取视频流并保存为图像序列和 MP4 视频文件。
- **主要逻辑/处理内容**:
  - **ROS 接口**: 使用 `rosbag` 库读取数据包，专门筛选 `/camera/color/image_raw/compressed` 话题。
  - **图像解码**: 使用 `cv_bridge` 和 `cv2` (OpenCV) 将压缩的 ROS 图像消息解码为 OpenCV 格式。
  - **文件输出**:
    - **图片**: 将每一帧保存为 `.jpg` 文件，文件名包含时间戳。
    - **视频**: 将图像序列合成并写入为 `output_video.mp4` 文件 (默认帧率 30 FPS)。
  - **自动管理**: 自动创建输出目录 `output_images`。

#### 2. 脚本名称：`cal_quar_params_qt.py`

- **文件类型**: Python GUI 应用程序 (依赖 PyQt5, NumPy)
- **功能概述**: 这是一个基于图形用户界面的数学计算工具，专门用于计算四元数 (Quaternion) 的演变。
- **主要逻辑/处理内容**:
  - **用户界面**: 提供输入框让用户输入角速度 ($w_x, w_y, w_z$)、持续时间 ($t$) 以及初始四元数 ($q_0, q_1, q_2, q_3$)。
  - **核心算法**: 根据输入的角速度和时间，计算旋转角度 $\theta$，利用四元数微分方程或旋转公式（涉及 $\cos(\theta/2)$ 和 $\sin(\theta/2)$）来求解新的四元数状态。
  - **结果显示**: 点击“Calculate”按钮后，在界面上直接显示计算后的新四元数。

#### 3. 脚本名称：`cpu_or_gpu_stress_test.sh`

- **文件类型**: Shell 脚本 (针对 Nvidia Jetson 平台)
- **功能概述**: 硬件压力测试脚本，用于对 CPU 或 GPU 进行高负载测试，同时监控系统状态（温度、功耗、频率）。
- **主要逻辑/处理内容**:
  - **环境检查**: 检查是否安装了 `stress-ng` (CPU 压力工具) 和 `gpu-burn` (GPU CUDA 压力工具)。
  - **交互菜单**: 提示用户选择测试模式（1 为 CPU 测试，2 为 GPU 测试）。
  - **后台监控**: 启动 `tegrastats` 工具并在后台运行，将系统硬件状态记录到 `stress_test_log.txt`。**注意：`tegrastats` 是 Nvidia Jetson 系列（如 Orin, Xavier, Nano）专有的监控命令。**
  - **负载执行**:
    - **CPU**: 运行 `stress-ng` 进行矩阵运算压力测试，持续 60 秒。
    - **GPU**: 运行 `gpu-burn` 进行浮点运算压力测试，持续 60 秒。
  - **清理**: 测试结束后自动终止后台的监控进程。

### 4 其他脚本：`start_camera_with_retry.sh`

- **功能概述**: **容错启动脚本**。专门用于启动 CSI 摄像头驱动，内置自动重试逻辑。若摄像头初始化失败或节点意外退出，脚本会自动尝试重启，确保视觉系统在无人机演示或测试中的稳定性。
- **启动的主要模块/指令**:
  - **核心驱动**: `roslaunch csi_camera_driver csi_camera.launch`。
  - **控制逻辑**: 包含监控进程状态的循环结构，当检测到节点退出时执行延时重试。
