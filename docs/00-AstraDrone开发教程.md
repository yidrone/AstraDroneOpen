# AstraDrone 开发教程（入门指南）

## 1. 前言
AstraDrone 是一个面向科研和教学的开源无人机平台，支持 ROS1 和 ROS2 两种系统，并提供从仿真到硬件的一体化方案。本教程旨在帮助初学者快速了解项目结构、搭建开发环境以及基本的gazebo仿真使用，同时理解无人机上位机与下位机的协同工作方式，并逐步掌握从基础操控到高级算法（如 Fast‑LIO2、EGO‑Planner、自主探索等）。

### 1.1 教程目的
- **快速入门**：面向零基础读者，介绍安装与运行步骤。
- **深入理解**：讲解系统架构，上位机与下位机的通信机制。
- **循序渐进**：分为仿真和硬件两大部分，从简单功能逐步提升到高级功能。

## 2. 系统总体架构
AstraDrone 采用典型的无人机分层架构：

1. **上位机（PC 端）**：运行仿真环境（Gazebo、PX4 SITL）、感知和规划算法（SLAM、EGO‑Planner 等），负责任务逻辑与可视化。上位机通常使用运行 Ubuntu + ROS 的 PC 或工控机，通过串口、有线网络或局域网与下位机通信。
2. **下位机（飞控与嵌入式端）**：包括飞行控制器（如 Pixhawk）、IMU、GPS、摄像头、激光雷达等传感器。下位机负责姿态解算、低级控制和基础传感器数据采集，通过 MAVLink 或 DDS 等协议与上位机交互。

系统中各功能模块的代码位于 `AstraDrone_ros1_ws/src` 目录。下面是各主要模块的简要说明：

| 模块 | 功能简介 |
|----|----|
| **Communication** | 通信接口，负责上位机与下位机的消息传输与协议适配。 |
| **Control** | 无人机控制逻辑，包括姿态控制、位置控制等。 |
| **MissionControl** | 任务调度模块，定义并执行飞行任务和状态机。 |
| **Planner** | 路径规划器，包含局部规划和全局规划算法。 |
| **SLAM** | 定位与建图模块，支持激光/视觉 SLAM（可扩展 Fast‑LIO2）。 |
| **Exploration** | 探索算法，结合 SLAM 与规划，实现自主探索未知环境。 |
| **Swarm** | 多机协同控制与通信。 |
| **Detection / Track** | 目标检测与跟踪相关算法。 |
| **Land** | 降落和着陆控制逻辑。 |
| **Utils** | 工具库、通用函数。 |

这些模块通过 ROS 话题、服务和动作进行协作，形成完整的无人机系统。

## 3. 开发环境与项目结构

### 3.1 环境准备

#### **3.1.1** pc端环境配置

​	支持双系统、虚拟机以及wsl等多种应用情形

1. **操作系统**：推荐使用 Ubuntu 20.04；支持 Ubuntu 18.04/22.04。

2. **ROS 安装**：根据需求选择 ROS1 (Noetic) 或 ROS2 (Foxy/Humble)。

3. **配置代理**：需要提前配置好代理，能够顺畅连接github，便于后续px4 submodule以及其他源码的克隆。

   这里提供clash for windows for linux安装网址：https://down.clashchinese.com/soft/clashchinese.com_Clash.for.Windows-0.20.39-x64-linux.tar.gz。后续可以直接解压，终端启用cfw即可。

   配置代理：在.bashrc中添加一下字段：

   ```shell
   alias proxy='export http_proxy="http://127.0.0.1:7890"; export https_proxy="http://127.0.0.1:7890"; export all_proxy="socks5://127.0.0.1:7890"; export no_proxy="localhost,127.0.0.1,::1"; echo "Proxy: ON"'
   
   alias unproxy='unset http_proxy; unset https_proxy; unset all_proxy; unset no_proxy; echo "Proxy: OFF"'
   ```

   在进行下述的安装之前，现在terminal中输入“proxy”，即打通了系统代理，可以克隆源码。

4. **安装脚本**：支持px4安装与初始化、相关源码编译与第三方库安装。

   ```terminal
   cd ~/AstraDroneOpen/ && sudo chmod 777 ~/AstraDroneOpen/scripts/*
   ./scripts/pc_installer.sh
   ```

   注意，在运行本脚本时一定要注意输入密码以授予sudo权限。

   注意，wsl运用该脚本时要事先apt install一下gnome-terminal，以便后续的安装。

5. **工作区初始化**：

   ```bash
   # 创建工作空间并克隆源码
   cd ~/AstraDroneOpen
   mkdir -p AstraDrone_ros1_ws/src
   cd AstraDrone_ros1_ws/src
   # 将仓库中的各模块拷贝/链接到 src 目录
   # 然后返回工作空间根目录编译
   cd ..
   catkin_make
   source devel/setup.bash
   ```

### 3.2 代码结构
仓库根目录下的关键文件/目录：

- `AstraDrone_ros1_ws` / `AstraDrone_ros2_ws`：分别用于 ROS1 和 ROS2 的工作区。内部 `src` 包含各功能包。 
- `docs`：教程和文档，包括本文件。 对项目各个部分的功能以及源码进行了介绍与讲解。
- `simulation`：仿真相关资源，如 Gazebo 世界文件、PX4 配置等。 
- `scripts`：安装脚本、编译脚本和运行脚本。 
- `hardware`：硬件设计资料，如 3D 设计、PCB。 
- `third_party`：第三方库源码，如 apriltag、Livox SDK、nlopt 等。 

理解这些目录有助于快速定位功能模块。

## 4. 无人机软件运行流程

1. 无人系统整体上电：下位机启动飞控固件；上位机启动进入ubuntu系统。
2. 通过 MAVROS（ROS1）或 micro-ROS/fastDDS（ROS2）等方式建立上位机与下位机的通信连接。上位机能够实时捕获下位机的状态以及相关数据，上位机也可以向下位机传递消息、调用服务。
3. 上位机程序启动，可以获取各个传感器的信息，并通过启动SLAM模块构建地图获取无人机或无人车的位姿估计。
4. 启用Planner，其可以根据当前地图和目标点生成轨迹（可使用 EGO‑Planner 等局部规划算法），同时Control模块中编写的控制器程序可以接收轨迹并计算控制指令，通过飞控驱动电机，实现飞行。
5. MissionControl模块可以依据预设任务或用户指令调度各模块运行，实现起飞、巡航、降落、探索等逻辑复杂的任务。

## 5. 仿真部分
仿真是学习无人机算法的最佳方式。本项目提供了完整的仿真环境，用于无风险地测试和调试。且项目结构安排合理，为后续的拓展提供了便利。

### 5.1 基础环境搭建

1. **Gazebo 世界选择**：在 `simulation/astra_gazebo_worlds` 目录下，提供 `cangku.world`、`forest.world`、`suv.world` 等仿真场景。选择所需场景并在启动文件中指定。 
2. **启动仿真**：

   ```bash
   cd ~/AstraDroneOpen
   source AstraDrone_ros1_ws/devel/setup.bash
   # 运行示例仿真脚本
   bash scripts/run_sh/pc_example.sh
   ```
   该脚本会启动 PX4 SITL、Gazebo 环境以及无人机控制相关节点，并在 RViz 中显示状态。
3. **可视化调试**：使用 RViz 查看无人机的位置、点云、轨迹等信息；使用 rqt_graph 检查话题连接。
4. **构建新的世界**：可以利用gazebo中已有的模型和启动的世界文件，将相应的模型拖入世界当中，保存到指定路径后就可以作为新的世界运行。

### 5.2 添加传感器

为了模拟真实环境，可以在模型中添加摄像头、激光雷达等传感器。需要提前编写或绘制传感器的模型文件、插件源码等相关文件。

1. **添加相应的模型文件**：将传感器本身的sdk以及模型文件添加到AstraSim部分，或者将其源码路径添加到GAZEBO_MODE_PATH当中，以便后续的仿真能够找到相应的模型以及其他插件。 
2. **修改 URDF/Xacro文件**：在机体描述文件中增加 `<sensor>` 元素，配置位姿，调整传感器的位置、碰撞体积以及相应的可视化参数。
3. **模拟驱动**：通过 Gazebo 插件模拟相机、IMU、激光雷达的发布。这通常是找到或者编写相应的SDK，编译后的结果。例如 `libgazebo_ros_camera.so`、`libgazebo_ros_laser.so`。 
4. **修改启动文件**：若模型的名字或者路径发生变化，或者需要同时启动多个实体，需要修改启动文件以供自己的需要。
5. **校准与验证**：在RViz中查看传感器数据，尽可能多的打开所需要的消息以及可视化信息检查坐标系是否正确，发布的话题是否有消息传输，相应的服务能否调用，同时检查调试发布频率和分辨率等。如点云数据需要在rviz中检查是否能够精
6. **启用相关程序**：传感器的消息传出以后，启用接受这些话题的节点，检测这些话题是否能够满足预期的效果。

### 5.3 基础功能示例

- **起飞与降落**：通过 MissionControl 中的服务调用或命令，或者启用下载好的地面站控制无人机起飞、悬停、降落。 

- **自主导航**：利用默认或者用户自定义的路径规划模块，在仿真环境中设置目标点，观察无人机的飞行路径和避障行为。 

- **数据记录**：使用 `rosbag record` 录制仿真过程，供后续debug。

### 5.4 进阶功能

进阶功能包括SLAM算法与Planner的必涨规划，以及自主探索等无人机的自主

#### 5.4.1 Fast‑LIO2
Fast‑LIO2 是一种快速、鲁棒的激光雷达‑惯性里程计算法，采用增量 kd‑Tree (ikd‑Tree) 实现高效的地图更新，支持 100 Hz 以上的激光数据处理，对不同类型 LiDAR 兼容性好，且可以直接基于原始点云进行匹配:contentReference[oaicite:0]{index=0}。使用步骤包括：

1. **编译与环境配置**：将 Fast‑LIO2 ROS 包添加到工作区中，按照其依赖安装和编译说明操作。 

2. **准备激光雷达数据**：在仿真中启用相关模型，确保传感器传出的话题名称、话题消息类型与 Fast‑LIO2 配置一致。 

3. **检查消息是否正确传递**：`info` 传感器信息，检查发布者订阅者是否正确，是否能够准确的传入FAST-LIO2本身。

4. **结果验证**：订阅里程计信息，若经过一段时间后里程计消息飘的厉害，需要检查PX4相应的参数是否设置正确以及算法本身是否存在问题。也可以在 RViz 中查看实时点云地图和位姿估计，评估精度和实时性。

5. **参数配置**：在上述过程调节完毕以后，根据科研或者其他任务需要调整地图分辨率、扫描频率等参数或者根据仿真模型调整相应的外参，使其适配仿真环境，并能够正确的反映实际需要。

#### 5.4.2 EGO‑Planner
EGO‑Planner 是一种基于梯度的轻量级局部规划算法，无需构建 ESDF 地图即可生成避障轨迹，规划时间约为 1 ms:contentReference[oaicite:1]{index=1}。使用步骤：

1. **集成 Planner 包**：在 `Planner` 模块中引入 EGO‑Planner 相关代码或配置。 
2. **启动方式**：在仿真环境中运行相应 launch 文件，确保 SLAM 模块为Planner算法提供位姿和地图信息。 
3. **结果验证**：在rviz中订阅planner相关的规划结果，如样条轨迹或者中间差值的规划点，是否能够准确绕过障碍物。
4. **调参**：根据无人机尺寸和速度需求调整安全距离、规划点数或加速度限制等参数。 在保证安全的前提下尽可能的适配要求。
5. **测试与调试**：设置障碍物场景，调整障碍物位置或者数量，观察局部规划轨迹是否平滑、安全。

#### 5.4.3 自主探索
在 Fast‑LIO2 和 EGO‑Planner 的基础上，可以实现自主探索功能：

1. **地图表示**：使用 OctoMap 或 voxel grid 存储未知区域和已知障碍。 
2. **目标选择**：利用前沿检测或采样策略选择下一个探索目标点。 
3. **路径规划**：调用 EGO‑Planner 生成到目标点的安全路径。 
4. **反馈机制**：当新的区域被探索后更新地图，再次选择新的目标，直至完成区域覆盖。 

## 6. 硬件部分
硬件实机操作是将仿真算法应用到真实无人机的关键。本部分介绍硬件搭建、上电流程、传感器接入以及高级功能的移植。

### 6.1 基础硬件操作

1. **硬件组成**：无人机通常由机架、分电板、电调、电机、螺旋桨、飞控（如 Pixhawk）、电源模块、机载计算机（如 NUC、Jetson）、IMU、GPS、相机、激光雷达等组成。 

2. **连线指导**：

   本部分不包括具体的动力部分的接线、请在px4官方文档或者相应商家、采用率高的帖子指导下进行。

   - **飞控与机载计算机**：通过 USB‑TTL 或以太网连接，实现 MAVLink 或 DDS 通信。
   - **传感器接入**：按官方文档连接相机、LiDAR、电源；注意线序和接口匹配。 
   - **电源分配**：确保传感器、电调和机载计算机供电稳定，建议使用 PDB 或专用电源模块。

3. **上电流程**：

   - 检查螺旋桨安装方向和紧固情况。 
   - 先接通飞控和传感器电源，等待初始化完成；再启动机载计算机和上位机软件。 
   - 使用地面站（如 QGroundControl）检查飞控状态、传感器校准和 GPS 锁定情况。 

4. **传感器标定**：进行 IMU 校准、磁罗盘校准、相机内参和外参标定、LiDAR‑IMU 外参标定等，提高算法精度。

### 6.2 实机调试流程

1. **固件配置**：为飞控刷入 PX4 或 ArduPilot 固件，设置飞行模式、PID 参数等。 
2. **单机悬停测试**：先在空旷场地进行悬停测试，检查姿态控制、传感器数据是否正常。 
3. **导航功能**：在机载计算机上运行定位、建图与规划算法，通过上位机发送目标点指令，观察无人机飞行。 
4. **安全措施**：配备急停开关，必要时设置返航点；调试过程中保持目视距离并随时准备手动接管。

### 6.3 高级功能移植

#### 6.3.1 Fast‑LIO2 实机运行

1. **硬件要求**：推荐使用支持高帧率激光雷达（如 Livox Avia、Velodyne）和高性能 IMU。 
2. **同步与标定**：确保 LiDAR 与 IMU 的时间同步，并完成外参标定。 
3. **运行步骤**：
   - 在机载计算机上安装并编译 Fast‑LIO2；按照文档配置 Livox 驱动和依赖。 
   - 修改 launch 文件，指定激光雷达话题、IMU 话题和外参矩阵。 
   - 启动 Fast‑LIO2 节点，实时查看建图结果，验证精度和延迟。 

#### 6.3.2 EGO‑Planner 实机运行

1. **环境准备**：确保有稳定的定位输入（来自 Fast‑LIO2 或其他 SLAM）；构建局部地图。 
2. **参数调整**：根据真实无人机尺寸、最大速度和加速度调整规划器参数；增大安全距离应对感知滞后。 
3. **调试过程**：
   - 首先在简单环境中调试避障功能，逐渐加入复杂障碍物。 
   - 记录飞行日志，分析规划轨迹与实际轨迹差异，优化控制器。 

#### 6.3.3 自主探索实机测试

1. **任务设计**：设置需要探索的区域边界和终止条件（如探索时间、覆盖率）。 
2. **在线地图更新**：实时融合点云和导航信息，维护占用栅格或 OctoMap。 
3. **前沿检测与目标选择**：运行探索算法选择未知区域作为新目标；通过 EGO‑Planner 生成路径。 
4. **安全机制**：考虑传感器盲区和动作延迟，设置避障冗余和终止条件。

## 7. 进阶与扩展

- **多机协同 (Swarm)**：使用 `Swarm` 模块实现多机编队、队形控制与避障，支持集中式和分布式通信。 
- **目标检测与跟踪**：通过 `Detection` 和 `Track` 模块，结合深度学习或传统算法实现目标识别与跟踪，可用于搜索救援或跟随任务。 
- **任务控制自定义**：修改 `MissionControl` 中的状态机，实现自定义任务逻辑，如拍照、投送物资等。 

## 8. 常见问题与调试建议

1. **依赖安装失败**：确认系统源是否可用，必要时更换镜像；逐个安装依赖并查看错误信息。 
2. **仿真不同步**：检查计算机性能是否满足要求；关闭不必要的节点；同步 Gazebo 与 PX4 的时间。 
3. **传感器数据异常**：排查硬件连接、驱动版本；在 RViz 中查看原始话题是否有数据。 
4. **地图漂移或轨迹抖动**：调整 SLAM 参数，检查传感器标定；避免震动和磁干扰。 
5. **规划失败**：确认局部地图是否正确构建；适当调整规划器的安全距离和动态窗口参数。
